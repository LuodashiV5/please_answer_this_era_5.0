
# 使用推荐的工具

## 搜索工具对比

|功能类别|应避免使用|推荐使用|主要区别|
|---|---|---|---|
|文件搜索|`find`|Glob 工具|Glob 提供更简洁的模式匹配语法，支持通配符模式|
|文本搜索|`grep`|Grep 工具 / ripgrep|推荐工具通常提供更好的性能和更友好的输出格式|
|任务搜索|手动查找|Task 工具|Task 工具提供结构化的任务管理和搜索功能|

## 文件读取工具对比

| 功能类别   | 应避免使用  | 推荐使用    | 主要区别                         |
| ------ | ------ | ------- | ---------------------------- |
| 查看文件内容 | `cat`  | Read 工具 | Read 工具可能提供更安全的文件读取和更好的格式化输出 |
| 查看文件开头 | `head` | Read 工具 | Read 工具统一接口，避免记忆多个命令         |
| 查看文件结尾 | `tail` | Read 工具 | Read 工具提供一致的使用体验             |
| 列出目录内容 | `ls`   | LS 工具   | LS 工具可能提供更结构化的输出和更多过滤选项      |

## 核心优势总结
**传统工具的局限性：** - 语法不统一，学习曲线陡峭 - 输出格式不够友好 - 缺乏智能过滤和上下文感知 - 性能可能不够优化
**推荐工具的优势：** - 统一的接口设计 - 更好的性能优化 - 智能过滤和上下文感知 - 更清晰的输出格式 - 更好的错误处理

## 使用建议
1. **搜索场景**：使用 Grep/Glob/Task 工具替代 find/grep
2. **读取场景**：使用 Read/LS 工具替代 cat/head/tail/ls
3. **性能要求高**：考虑使用 ripgrep 替代传统 grep
4. **项目管理**：使用 Task 工具进行任务组织和搜索
