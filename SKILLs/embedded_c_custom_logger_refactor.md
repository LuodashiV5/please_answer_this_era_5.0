# Embedded C Custom Logger Refactor (EasyLogger 版)

本 Skill 用于将嵌入式代码（特别是针对 Renesas/ARM 等平台）中的 `printf` 或分散日志重构为符合 **EasyLogger** 工业级标准的规范代码。特别融入了来自 `app_vsfan_ctrl_thread_entry.c` 的实战经验。



## ⚠️ 核心准则 (Hard Constraints)

1.  **标签化管理**：每个 `.c` 文件顶部必须定义 `LOG_TAG`，并紧随其后包含 `#include <elog.h>`。
2.  **禁止直连 API**：严禁直接调用 `log_i()`/`log_e()`。业务代码必须通过定义的模块封闭宏（如 `{{MOD}}_LOG_I`）进行解耦转换。
3.  **双重开关机制 (Double Switch)**：每一个日志输出点都必须受控于：
    *   **编译级开关**：`#define APP_{{MOD}}_LOG_ENABLE`（量产环境通过置 0 来彻底消除代码体积）。
    *   **运行时开关**：`volatile uint8_t app_{{mod}}_log_switch`（支持在线实时动态控制输出）。
4.  **严禁 printf**：自动识别并将所有 `printf` 及其变体替换为对应的封装宏。
5.  **精简原则**：单条日志尽量控制在 128 字节内，禁止过度啰嗦。推荐使用缩写（Init, Err, FSM）并附带十六进制错误码。
6.  **分级策略**：
    *   **DEBUG**: 状态机迁移细节、高频传感器内部数值采样。
    *   **INFO**: 初始化状态、关键动作发起。
    *   **WARN**: 阈值接近、非致命重试（如传感器异常采样后的次重试）。
    *   **ERROR**: 函数返回错误码、硬件失效、流程中断。
7.  **性能禁区**：禁止在 PWM、定时器中断或 1ms 级的底层闭环回路中插入日志。

## 📥 输入变量 (Variables)

*   `{{source_code}}`: 待处理的原始代码。
*   `{{module_name}}`: 模块名（会自动处理为大写 `{{MOD}}` 用于宏，小写用于 `LOG_TAG`）。
*   `{{log_level_pref}}`: 日志初始开启等级。

## 🛠️ 处理逻辑 (Workflow)

1. **Header 封装生成**：在对应的头文件或源文件头部生成如下双重保护宏定义：

    ```c
    #define APP_{{MOD}}_LOG_ENABLE    1  // 编译总开关
    
    #if APP_{{MOD}}_LOG_ENABLE
      extern volatile uint8_t app_{{mod}}_log_switch;
      #define {{MOD}}_LOG_ENABLED (app_{{mod}}_log_switch == 1)
      #define {{MOD}}_LOG_E(...) do { if ({{MOD}}_LOG_ENABLED) log_e(__VA_ARGS__); } while(0)
      #define {{MOD}}_LOG_W(...) do { if ({{MOD}}_LOG_ENABLED) log_w(__VA_ARGS__); } while(0)
      #define {{MOD}}_LOG_I(...) do { if ({{MOD}}_LOG_ENABLED) log_i(__VA_ARGS__); } while(0)
      #define {{MOD}}_LOG_D(...) do { if ({{MOD}}_LOG_ENABLED) log_d(__VA_ARGS__); } while(0)
      #define {{MOD}}_LOG_V(...) do { if ({{MOD}}_LOG_ENABLED) log_v(__VA_ARGS__); } while(0)
    #else
      #define {{MOD}}_LOG_E(...) ((void)0)
      #define {{MOD}}_LOG_W(...) ((void)0)
      #define {{MOD}}_LOG_I(...) ((void)0)
      #define {{MOD}}_LOG_D(...) ((void)0)
      #define {{MOD}}_LOG_V(...) ((void)0)
    #endif
    ```

2. **变量实例化**：在 `.c` 文件顶部定义运行时控制变量：`volatile uint8_t app_{{mod}}_log_switch = 1;`。

3. **智能转换注入**：
    *   识别 `if (ret != OK)` 等异常判断，注入 `{{MOD}}_LOG_E` 记录错误详情。
    *   在函数入口/重大逻辑转向处（如 FSM 切换）注入 `{{MOD}}_LOG_I` 或 `{{MOD}}_LOG_D`。

4. **文本压减**：自动将长句转化为类似 `[Mod] Action Failed: 0x%02X` 的精简格式。
