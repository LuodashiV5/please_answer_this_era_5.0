# C语言核心重构模式：数据隔离与面向对象

这份指南提炼了将离散的、面向过程的C代码重构为高可维护性结构的核心思想。该模式尤其适用于嵌入式系统、固件开发以及复杂计算逻辑模块。

## 1. 核心特征总结

*   **上下文结构体封装上下文**：所有需要引用的变量不再是杂乱无章的全局变量，也不再是函数之间传来传去的长参数列表，而是全部装入一个结构体（Context / Object）。
*   **变量前缀约定数据流向**：强命名规范，一眼辨识变量是外部送进来的，还是我们要送出去的。
*   **IPO（输入-处理-输出）管线隔离**：彻底解耦“总线数据获取”与“纯业务计算”。业务逻辑应当对“数据来自哪里”一无所知。

## 2. 重构步骤细节

### 步骤一：提取上下文并明确I/O方向
创建一个表示模块本身状态和输入输出的结构体（例如 `module_name_t`）。
*   **输入变量（Inputs）**：必须加上 `in_` 前缀。它们代表计算依赖的只读数据。
*   **输出变量（Outputs）**：必须加上 `out_` 前缀。它们代表模块对外发布的运算结果。
*   **内部状态（Internal States）**：不加前缀，用于维护周期计算需要的计数器或历史值。

```c
typedef struct {
    // 可选的面向对象方法绑定
    void (*Do)(void* pThis); 
    
    // 输入数据 (通过 _read 从外部读取)
    float in_sensor_temp;
    bool  in_enable_flag;
    
    // 输出记录 (通过 _write 写回到外部)
    float out_target_value;
    
    // 内部状态
    int   calc_period_cnt;
} module_name_t;
```

### 步骤二：强制实施 "读-算-写" (Read-Calculate-Write) 管线
将业务的执行（如定时器任务的Tick入口）严格切分为三个互不干涉的阶段。计算逻辑函数中**严禁**直接调用获取或写入外部资源的API（如 `RD_DB_...`, `WR_DB_...`）。

```c
static void _do(module_name_t *pThis)
{
    // 阶段1：数据采集 (隔离外部信源)
    _read(pThis);
    
    // 阶段2：纯算数与业务逻辑判断 (内部闭环计算)
    _calc_step_1(pThis);
    _calc_step_2(pThis);
    
    // 阶段3：数据下发 (隔离外部信宿)
    _write(pThis);
}
```

### 步骤三：隐藏底层数据存取细节
在 `_read` 和 `_write` 函数内部，再负责与真正的系统总线、数据库或者全局变量打交道。
这样的好处是：即便底层 RTOS 或硬件平台的 API 变更，也只需要修改这两个薄薄的封装函数，核心的算法文件 `_calc_xxx` 可以做到一行不改。

```c
static void _read(module_name_t *pThis)
{
#if _UNIT_TEST_MODE_
    // 单元测试时，从CSV文件或本地Mock结构体读数据
    read_mock_data(pThis);
#else
    // 实际运行时，从系统数据库或全局变量读数据
    pThis->in_sensor_temp = RD_DB_FLOAT(TEMP_SENSOR_ID);
    pThis->in_enable_flag = RD_DB_BOOL(ENABLE_FLAG_ID);
#endif
}
```

### 步骤四：提供抽象的工厂化创建
使用 `xxx_create()` 工厂方法，为结构体分配内存（堆或静态池都可以），并必须妥善赋予所有内部和输出变量**安全的初值**，并绑定方法指针。

## 3. 为什么要这样重构？（核心收益）

1.  **极佳的单元测试友好度**：因为计算过程不碰外部硬件接口，只需将 `_read`/ `_write` 用条件编译替换为读取文本文件，就可以在 PC 上（脱离实际开发板）对核心代码写测试用例，传入指定的 `in_` 值，断言预期的 `out_` 值。
2.  **避免状态竞争**：所有的计算都在这一帧采集的“数据快照”(`in_` 变量)中进行，同一帧内数据不会突然因为外部中断的变化而变化，消除了大量玄学 Bug。
3.  **认知负担低**：新员工想了解此模块依赖哪些外部数据，只需看结构体有多少个 `in_` 变量即可，所有耦合关系在头文件里一目了然。
