上篇文章我提到我创建了一个"内容平台分析 skill"，今天我就分享一下如何用Skill Creator创建这个技能的完整过程。

之前我用一个公众号文章下载的工具导出了几个对标账号的所有文章数据，拿到了**几百个Markdown文件 + 一张包含阅读量、点赞、分享等所有互动数据的Excel表格**。

（还不了解这个工具的，可以看看我的上篇文章[如何一键导出任意公众号历史文章数据并用 Agent 拆解分析（保姆级教程）](https://mp.weixin.qq.com/s?__biz=MzkzNTk3NTYyNA==&mid=2247484215&idx=1&sn=016f7a7b0f76eb6a487d92cd73e41678&scene=21#wechat_redirect)）

数据有了，但问题来了。

每次要分析新的账号，都得跟AI说："帮我分析标题、开头、结构、话题、爆款特征，整理成报告，再和我的账号对比..."

第一次还行，第二次、第三次...每次都要从头解释，质量还不稳定。有时候分析得特别细，有时候又漏掉关键点。

而且我发现，**我自己其实有一套固定的分析思路**：先看发文频率和长度，了解基本情况；然后研究爆款的标题和开头，找规律；再分析话题分布；最后对比差距。

但是每次都要手动告诉AI怎么执行，太累了。

我就在想：**能不能把这套分析流程"固化"下来，让AI自动按照我的思路去分析？**

---

## 用Skill Creator创建技能

前段时间我刚好在研究 Agent Skills，发现有个叫 **skill-creator** 的技能，专门用来帮你创建新的技能。

它的思路很简单：你告诉它你想做什么，它会帮你分析任务流程、拆解成清晰的步骤、设计好输出格式、生成完整的技能文件。

之前我已经在opencode中基于原始数据进行了深度分析并产出了分析报告，我觉得这个流程是可复用的，于是让它分析拆解我们之前的分析流程文档，看看还有哪些需要优化和改进的地方，并且进行了调研。

再系统性思考这个 Skill 该如何设计。最后再调用skill-creator创建技能。

然后它就开始了。

![Image](https://mmbiz.qpic.cn/mmbiz_png/gicwqF173AkNgQwxlCL3keknRZsrHtJjxZibMV1zJiaQ9VKicpU09KngOuicVKjpiaKI8ib8WgJ93B0TPHJWibPLonUFscJ4p5Ixdg7G1biaErhRnltI/640?wx_fmt=png&from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=0)

首先，它让我提供一些分析流程的参考文档。我就把我平时分析账号时的一些思考过程、分析框架写给了它。

然后它开始拆解任务。

它没有急着生成技能文件，而是先问了我几个问题：核心目标是什么？需要分析哪些维度？报告应该包含什么内容？用户会提供什么数据？

我一一回答后，它开始设计流程。

它把整个分析流程拆成了4个核心步骤：

**第1步：内容理解** - 先读懂内容本身：发布频率、爆款模式、主题分析、内容类型、写作手法

**第2步：数据验证** - 用数据证明内容理解的有效性：相关性验证、时间趋势、分位数分析

**第3步：对标分析** - 对比对标账号与自己账号：内容深度、写作技巧、数据表现、优势劣势

**第4步：建议生成** - 提供可执行的具体建议：标题优化、内容结构、写作技巧、内容策略

每个步骤都设计了具体的分析维度和输出格式。

## 快速迭代优化技能

大概过了十几分钟，第一版技能生成了。

我迫不及待地测试了一下："调用content-platform-analyzer技能，数据在XX文件夹"

确实生成了报告，但是...一看，有点失望。

![Image](https://mmbiz.qpic.cn/mmbiz_png/gicwqF173AkOKPJ9pWha5L4Lia4KajPEjamG6cX2Bibk0Xy9Gical2rCiaRwM4PFraTDozwKcTulsJSqIk9Bg7RNZkqGS0xU7LSW15fB7870sWBA/640?wx_fmt=png&from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=1)

报告太简略了，才200多行。它确实分析了一些基本数据，但是对于一些我非常关心的细节，分析得很浅。

我跟它说："这个报告太简略了。我需要更细致的分析：标题分析不仅要分类，还要给出具体的标题模式和示例；开头钩子要总结出具体的钩子类型，比如'数字悬念型'、'痛点共鸣型'等；段落结构要分析具体的写作套路；爆款文章要单独列出，逐篇拆解。"

这一次，它重新设计了分析框架，把每个维度的分析做得更细致。

标题分析增加了模式分类和示例统计，开头钩子设计了类型识别和效果评估，爆款文章要求逐篇拆解。

![Image](https://mmbiz.qpic.cn/mmbiz_png/gicwqF173AkPT6Gw4hiajQqPacMiatRALVBAmDO7B5FNafibKyIzu3pTgzqBPiao4BB9GQeL8RgVYichIgp8rXGwLWibc2c2Rj5c4vL7c8GXo9QUZs/640?wx_fmt=png&from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=2)

  

报告长度确实提升了，但用了几次后，我发现还是不太对劲。

![Image](https://mmbiz.qpic.cn/sz_mmbiz_png/gicwqF173AkMOrSxHkj0cHjiaqrknCpNEr9l2v4MLdiaws5kInNKkArt6PRYSBRk3zdjs1q2xkLiaXPM8nsHUYFzafoaC8YGtEepYs7PK9dfzX0/640?wx_fmt=png&from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=3)

报告虽然详细了，但过度聚焦在写作技巧上。标题、开头、结构、结尾这些占了大量篇幅，但是**内容策略、时间序列、根因分析、定位、路线图**这些战略层面的内容几乎没有。

我又做了一轮更深入的优化。这次调整幅度很大：

**减少写作技巧部分**（从40%降至20%）：删除了emoji使用规范、结尾CTA这些过度细节。

**新增6个战略层级部分**（150-200行，占60%）：内容策略分析、内容演进轨迹、时间序列分析（爆发月份、趋势拐点）、根因分析框架（表象→根因三层分析）、战略定位与差异化（3-Lens框架）、12个月增长路线图（3阶段）、立即行动的杠杆点。

这次优化后，报告从"术"（写作技巧）转向了"道"（战略思考）。

最终报告模板从759行扩展到1037行，内容结构变成了：20%写作技巧 + 60%战略分析 + 20%数据/模板。

现在用起来，效果完全不一样了。

我要分析一个新的对标账号，只需要说："调用 Content Platform Analyzer 技能，分析这个账号：数据路径在XX"

然后它就会自动执行4个步骤的分析，生成一份详尽的战略级报告。整个过程大概3-5分钟，而我只需要等结果就行。

## 这个思路可以复用到很多地方

其实不只是内容分析。

我们日常工作中，有多少事情是需要反复做的？

- **数据分析**：每周要做的数据周报，分析维度都是固定的
    
- **竞品调研**：每次调研新产品，都要看产品功能、商业模式、用户评价等
    
- **代码审查**：每次Review代码，都要检查安全性、性能、可维护性
    
- **文章写作**：每次写公众号，都要考虑标题、开头、结构、结尾
    

这些重复性的工作，都可以尝试用 Skill 固化下来。

**把你的工作经验"固化"成可复用的流程，不是每次都临时教AI怎么做，而是把积累的经验沉淀成技能包。**

这样做的好处是：

1. **一次编排，反复使用**。你花时间打磨的分析框架，可以持续复用
    
2. **质量稳定可控**。AI会严格按照你的框架执行，不会遗漏关键点
    
3. **可以分享给别人**。技能本质就是一些文档和脚本，别人装上就能用
    
4. **持续迭代优化**。每次使用后发现不足，可以不断改进
    

## 写在最后

现在回过头来看，**学会创建Agent Skill，真的是一件投入产出比很高的事情**。

它让我从繁琐的重复劳动中解放出来，把精力放在更有价值的思考和创作上。

而且创建Skill的过程本身也很有价值 - 它逼着你把模糊的"经验"变成清晰的"流程"，把"我大概知道怎么做"变成"我能准确描述每一步该做什么"。

这种对工作方法的系统性思考，本身就是一种成长。

如果你也有反复在做的事情，不妨试试把它编排成一个Skill。

一开始可能觉得麻烦，但长期来看，这是在给自己打造省力的工具。

把重复的工作交给AI，让自己有更多时间去做更有价值的事情。
