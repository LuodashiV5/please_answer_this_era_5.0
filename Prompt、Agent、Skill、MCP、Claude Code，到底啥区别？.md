最近经常有人问：这几个词到底啥关系？今天一篇说清楚。  
  
一句话总括：Prompt是“口头指令”，Skill是“SOP手册”，MCP是“USB接口”，Agent是“实习生”，Claude Code是“实习生的电脑” 。  
  
- 分别是什么？  
  
Prompt：你给AI的“口头交代” 临时指令，用完即走  
Agent： 能“做事”的AI 自主规划+调用工具  
Skill ：AI的“岗位说明书+工具箱” 可复用的标准化能力包  
MCP ：AI连接外部世界的“USB接口” 统一协议，连接数据/工具  
Claude Code： 给AI用的“电脑终端” 执行环境，让AI操作真实系统  
  
- 为什么这么分？  
  
传统做法是把所有规则塞进Prompt，结果上下文爆炸、模型混乱 。  
  
现在的分层逻辑更清晰：  
  
· Agent负责“想”：理解目标、拆解任务、决定下一步  
· Skill负责“做”：封装好的标准流程，按需加载，节省Token  
· MCP负责“连”：让AI能统一调用数据库、API、文件系统  
· Claude Code提供“场”：给AI一个真实的操作环境  
  
- 怎么用？  
  
一个真实工作流说明一切 ：  
  
1. 你告诉Claude Code：“帮我分析这份财报”  
2. Agent拆解任务：需要提取数据→分析趋势→生成图表  
3. 自动匹配Skill“财报分析技能包”（含分析规则、模板）  
4. 通过MCP连接数据库和绘图工具  
5. Claude Code在终端里执行所有操作  
  
你看到的只是结果，背后这套分工已经标准化。  
  
✨ 最后说一句  
  
Prompt是起点，但AI真正进入“工作流”，靠的是这套分层体系。口头交代变成SOP，临时调用变成模块复用——这才是AI从“玩具”到“工具”的关键跃迁 。
