# OpenClaw 装好了，但不知道干啥？先别泄气，看看别人在解决什么问题

 

[绿蚁红泥天欲雪](javascript:void(0);)

 

装了 OpenClaw，配了 soul.md，跟它聊了几句，觉得"嗯，挺聪明的"——然后就没有然后了。

工具安装前是最兴奋的，而接下来才是漫长的迷茫期：不知道用它做什么。

刷了一圈教程，全是教你怎么装、怎么配。但装完配完之后呢？

面对着 3000 个 Skill，却很少有将想法落实的，而想法都散落在 Reddit 和 X 上。

最近看到一个 GitHub 仓库 awesome-openclaw-usecases，他把 22 个实际跑过的 OpenClaw 用例，整理成了一个 awesome 仓库，从刷社交媒体到管服务器什么都有，甚至还有搞量化交易的。

翻完之后我觉得，这些解决了实际痛点的用例才是最值得看的，故在此做分享和学习。



> ## 最好的用例从痛点开始

这 22 个用例基本都是很小的一个功能，解决了自动化的问题，节省了时间。

我们"装了就吃灰"的根因，不是工具不行，是还没找到那个让我们烦透了的问题。

你在拿着锤子找钉子，但真正该做的是先感受到疼。所以，想别着急，想思考和回忆自己工作与生活中的小痛点，从一个具体的小场景开始入手，做出实现，然后迭代。

我也从中挑了 4 个聊聊。选得比较主观，就是翻的时候觉得"哎这个有意思"的。

> ## 用例赏析

> ### Daily Reddit Digest

每天刷 Reddit 半小时，真正有用的信息不到 5 分钟，剩下全是 meme 和口水仗。

这个用例干的事很直白：装一个 reddit-readonly 的 skill，告诉 OpenClaw 你关注哪些板块、什么内容不想看，它定时抓取热帖给你推一份精选摘要。它还有个反馈循环，每天推送完会问你"今天的内容怎么样"，反馈存进 memory，下次就更准。

换成 Hacker News、行业公众号、竞品动态，一样的逻辑。我自己倒是想拿它来过滤公众号，每天早上打开订阅号一堆红点，大部分点进去看两眼就退了，但又怕漏掉什么。让 OpenClaw 先帮我扫一遍，只把跟我关注方向相关的推过来，这事想想就舒服。

> ### Health & Symptom Tracker

你有没有过吃完烧烤，第二天肚子不舒服，但死活想不起来，你吃的是蝉蛹还是知了？

你想着"记个饮食日记"，但是坚持了三天就放弃了。这个用例就是解决这事的：通过 Telegram 记录每天吃了啥、身体啥反应，AI 自动找关联性。跟"技术"毫无关系，纯生活痛点，但是就是流水账。

做法朴素得不能再朴素，每天三个时间点（早 8 点、午 1 点、晚 7 点），OpenClaw 通过 Telegram 提醒你"吃了啥"，你回一句话，它记到 markdown 文件里。每周日自动跑一次分析，找食物和症状之间的关联。

一个 cron + 一个 Telegram topic + 一个 markdown 文件，齐活。

我看完第一反应是想给我自己弄一个，毕竟我的肠胃是真脆弱，容易拉肚子，但每次都是拉的不明不白的😂。健身打卡、情绪日记、用药记录，都是同一种套路。AI 不见得多聪明，但它比你有耐心，也比有记性。

> ### Personal CRM

维护人脉这件事吧，所有人都知道重要。几乎没人能坚持手动做。

这里的做法是让 OpenClaw 自动扫描你的邮件和日历，发现新联系人就建档，开会前自动给你准备一份 briefing——这人是谁、上次聊了什么、需要跟进什么。每天早上 6 点 cron 触发扫描，新联系人写进 SQLite，老联系人更新互动记录。7 点再跑一次，针对今天的会议准备参会人背景。

你还能随时通过 Telegram 问它"我上次和张三聊了什么""谁需要跟进了"，直接查库回答。

说白了就是解决一类事：你知道该做，但因为太琐碎所以永远不会手动做。

> ### Self-Healing Home Server

这个大多数人不需要复刻，但值得看。

一个 24/7 运行的基础设施 Agent，持有 SSH 权限，自动巡检家庭服务器、修复问题、做安全审计，每天早上 8 点给你发一份"晨报"。Kubernetes、Terraform、Ansible、ArgoCD 全串在一起，定时任务从 15 分钟一次的看板检查到每周一次的安全审计，层层嵌套。

但最值得看的不是它多炫，而是作者反复强调的一句话：

> "AI assistants will happily hardcode secrets."

翻译成人话：AI 会开开心心地把你的密钥硬编码进代码里。

所以他搞了一整套防线：pre-push hook 扫密钥、本地 Gitea 先过一遍再推远端、分支保护必须人工 review、1Password 专用 vault、每日自动安全审计。自动化的天花板，取决于你的安全地板。权限给多大，防线就要多厚。

> ## 回头看这些案例

这四个用例看着不一样，但用到的模式翻来覆去就那几种。信息聚合，从多个源抓内容过滤噪音推给你，Reddit Digest 是这个，换成公众号或行业网站也一样。定时触发，到点了自动干一件事，CRM 每天 6 点扫邮件是这个，你每天固定要做的重复性动作八成都能套。记录分析，你负责扔数据 AI 负责找规律。系统联动把几个工具串起来，Self-Healing Server 是极端例子。而且它们能组合着用，"定时触发 + 信息聚合"就是 Reddit Digest，"定时触发 + 记录分析"就是 Health Tracker。

你不会因为买了一把好菜刀就突然会做菜，你得先饿。但饿了之后翻翻菜谱，至少知道从哪道开始。上面这四种模式就是菜谱。

也没有谁一上来就搞复杂架构，Health Tracker 就是一个 cron 加一个 Telegram topic 加一个 markdown 文件。先跑起来再说。

复杂的系统都是从简单的功能开始的，最后之所以复杂，那是使用+迭代的结果。

> ## 找到你的第一个 use case

别再问"OpenClaw 能做什么"了。换个问法："我每天在重复什么？""什么事我知道该做但就是懒得做？"

花点时间复盘回想下，把你最烦的那件重复性小事写下来，然后翻翻这个 awesome list（仓库地址：https://github.com/hesamsheikh/awesome-openclaw-usecases），看看有没有现成的套路可以用。

不是工具在等一个更厉害的用户，而是你在等一个足够烦你的问题。但是问题需要你去主动的挖掘，还是那句老话“AI 时代最缺创意“。
