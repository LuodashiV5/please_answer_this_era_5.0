你现在做的都是 demo，接下来目标是：**从 demo → 能在公司 / 项目里真正用起来**。

# 给我推荐一些适合嵌入式工程师学习的AI工具

按你嵌入式老工程师 + 已用 Copilot/Cloud Code/Agent/RAG 的背景，我把 AI 工具分成**日常编码、本地 / 离线、RAG/Agent、边缘部署、工程化**五大类，每类只给最适合你的、能立刻落地的工具，不堆垃圾。

### 一、日常编码 AI 助手（你天天用，升级用）

#### 1. GitHub Copilot（你已用，必升级）

- 核心：**IDE 深度集成**（VS Code/CLion/Keil），实时补全、重构、注释
- 适合：驱动 / MCU/RTOS 代码、寄存器配置、中断、DMA、HAL/LL 库
- 进阶：**Copilot Chat + Cloud Code**，直接在 IDE 里查手册、生成测试、查编译错误
- 必开：**Copilot Skills**（你已写二三十个，继续扩：寄存器速查、时序生成、错误码解析）

#### 2. DeepSeek-Coder（嵌入式代码神器，替代部分 Copilot）

- 特点：**C/C++/ 汇编 / 嵌入式特化**，理解芯片手册、寄存器、时序、RTOS
- 优势：**长上下文、低幻觉、代码质量高**，适合复杂驱动、协议栈、算法
- 用法：本地部署（Ollama）+ VS Code 插件，**内网可用**
- 场景：STM32/ESP32/ARM 驱动、Modbus/CAN/USB 协议、PID / 滤波、内存优化

#### 3. Aider（终端 AI 结对编程，嵌入式强推）

- 特点：**理解整个代码库**，跨文件修改、自动适配 CMake/PlatformIO、交叉编译配置
- 适合：**重构、批量修改、驱动框架生成、测试用例**
- 用法：终端输入自然语言，直接改代码、编译、调试，比 Copilot 更 “懂项目”

#### 4. Kimi（长文档阅读，芯片手册杀手）

- 特点：**上传整本数据手册 / SDK 文档**，精准查寄存器、时序、电气参数
- 适合：**读 datasheet、查 Errata、对比芯片差异、长代码 Review**
- 用法：上传 PDF，问 “STM32H7 SPI 时钟配置步骤”“I2C 时序图解析”

------

### 二、本地 / 离线 AI 工具（嵌入式人核心优势：内网 / 涉密可用）

#### 1. Ollama（本地模型一键部署，必学）

- 特点：**一行命令跑本地大模型**，支持 Qwen-Code、DeepSeek-Coder、Llama 3、Phi

- 适合：**内网、离线、涉密环境**，完全不联网

- 必跑模型：

    - **DeepSeek-Coder 6.7B**：嵌入式代码生成
    - **Qwen3-1.5B/7B**：轻量、快、中文好
    - **CodeLlama 7B**：C/C++ 强

    

- 用法：`ollama run deepseek-coder:6.7b`，本地 RAG/Agent 全离线

#### 2. LlamaEdge（边缘 / 嵌入式本地推理）

- 特点：**极小内存占用**，跑在树莓派 / ARM/MCU 上，支持 GGUF 量化
- 适合：**端侧小模型、本地助手、离线 RAG**
- 优势：比 Ollama 更轻，适合资源受限设备

#### 3. 本地代码模型精选（按内存选）

- 1GB 内存：**Qwen1.5-1.8B-Code**、**DeepSeek-Coder 1.3B**
- 2–4GB：**DeepSeek-Coder 6.7B**、**Qwen3-7B-Code**
- 4–8GB：**CodeLlama 13B**、**DeepSeek-Coder 33B**（服务器级）

------

### 三、RAG/Agent 工具（你已做 demo，工程化必备）

#### 1. RAG 框架（嵌入式知识库）

- **LangChain**：最通用，支持向量库、文档分块、重排、工具调用
- **LlamaIndex**：**RAG 特化**，检索质量更高，适合嵌入式文档（手册 / SDK / 驱动）
- **Chroma**：轻量向量库，本地部署，适合小项目
- **Qdrant**：高性能向量库，适合大规模嵌入式知识库
- 必做：**嵌入式专属 RAG**（芯片手册 + SDK + 驱动 + 协议 + 你的代码库）

#### 2. Agent 框架（把 Skills 变成自动化工具）

- **AutoGPT**：基础自主 Agent，适合简单任务
- **LangGraph**：**可控、稳定、工程化**，适合嵌入式开发 Agent（查手册→写代码→编译→调试）
- **OpenClaw**：你已用，继续扩：**嵌入式开发 Agent**（自动查寄存器、生成驱动、检查 Bug）
- **CrewAI**：多 Agent 协作，适合复杂项目（代码 + 测试 + 文档）

#### 3. 工具调用（Function Calling，嵌入式核心）

- **OpenAI Function Calling**：Copilot 内置
- **DeepSeek Function Calling**：本地可用
- **LangChain Tools**：封装你的嵌入式工具（寄存器查、时序生成、编译、烧录）

------

### 四、边缘 AI 部署工具（嵌入式 ×AI，最值钱）

#### 1. 端侧推理框架（跑在 MCU/ARM/ 嵌入式 Linux）

- **TensorFlow Lite Micro（TFLite Micro）**：**MCU 首选**，支持 ARM Cortex-M、ESP32、STM32，极小内存（<32KB）
- **NCNN**：腾讯开源，轻量、快，支持 ARM/x86/RISC-V，适合嵌入式 Linux
- **MNN**：阿里开源，端侧优化，支持量化、剪枝
- **ONNX Runtime**：通用，支持多平台，适合模型转换
- **Tengine**：国产，针对 ARM/NPU 优化，适合工业嵌入式

#### 2. 模型转换 / 优化（把大模型变小，跑在板子上）

- **ONNX**：模型中间格式，统一转换
- **TensorRT**：NVIDIA GPU/Orin 优化，加速推理
- **RKNN**：瑞芯微 NPU 专用
- **量化工具**：GGUF（Ollama）、INT4/INT8（TFLite）、FP8（Qwen3）

#### 3. 边缘 AI 开发平台（零代码 / 低代码）

- **Edge Impulse**：**嵌入式 AI 神器**，自动采集传感器数据、训练模型、生成 TFLite Micro 代码，支持 MCU/ESP32/STM32
- **Google Coral**：TPU 加速，适合视觉 / 语音
- **NVIDIA Jetson**：高算力边缘，适合复杂 AI（视觉 / 语音 / 多模态）
