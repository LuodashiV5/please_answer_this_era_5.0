# SKILL: Code Review（日常通用）

> 给 Claude Code 使用：对我贴出的代码做日常 code review，偏工程可维护性 + 常见缺陷，仍然优先抓致命风险。

## 目标
在不啰嗦的前提下，指出：
- 致命风险（越界/溢出/空指针/竞态/UB）
- 常见缺陷（错误处理、可读性、可测试性、命名、注释、重复逻辑）
- API/模块边界问题（输入校验、返回值、资源释放）

## 输出（必须严格遵守）
1) **只提改善建议**（不要解释过程）。
2) **代码旁中文旁注**：
   - `// REVIEW[BLOCKER]` 致命
   - `// REVIEW[MAJOR]` 高风险/易踩坑
   - `// REVIEW[MINOR]` 风格/可维护性
3) **建议要可执行**：每条建议给出 *改哪里/怎么改*。
4) **优先“最小改动”**，其次才是重构建议。
5) 如果发现“可能是约定/外部保证”的地方：
   - 仍然提示风险，并建议用 `assert`/`static_assert`/边界检查把假设固化。

## 检查清单（输出时不必罗列清单本身）
- 输入合法性（长度、范围、空指针、枚举值）
- 错误处理一致性（返回码、日志、重试）
- 资源管理（锁、内存、句柄、外设状态）
- 并发安全（ISR/任务共享、volatile、原子性）
- 数值安全（位移、符号扩展、类型提升、浮点精度）
- 可读性（复杂条件拆分、早返回、命名、注释）

## 输出示例（格式示意）
```c
if (len > 256) { ... } // REVIEW[MAJOR]: len 上限与缓冲区容量不一致，存在越界写。修复：改为 len <= BUF_SZ，并统一用 BUF_SZ。
```
