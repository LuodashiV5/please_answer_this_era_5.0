


# Sirius & MBC 实验室准入 & 调试一体化 Checklist



## 文档目的说明

本文档旨在将 **Sirius / MBC 进入实验室与调试阶段的工程前置条件**，与 **实际可执行的操作步骤**进行统一整理与对齐。
 文档不再单独区分“要求说明”和“操作手册”，而是以 **What + How** 的方式展开：

- **明确每一步“必须做什么”**（工程结论、前置条件、不可跳过的要求）；
- **同时给出“如何实操”**（基于 debug tool / MBC 的具体参数设置、操作顺序与判据）。

通过该文档，工程人员可以在现场或实验室环境中：

- 快速判断**当前阶段该执行哪一步**；
- 明确在 **debug tool / MBC 上应如何配置与操作**；
- 避免因步骤缺失、顺序错误或参数未写入 Flash 等问题导致的反复排查。

本文档面向 **调试、校准、试装与实验阶段**，重点解决“知道要做什么，也知道如何把它正确做完”的问题。

## 0. 系统架构速览

| 项目       | 说明                                            |
| -------- | --------------------------------------------- |
| 控制频率     | 20 kHz（Ts = 50 μs），`main_cpu01.c:53`          |
| 轴承配置     | 径向 4 通道（X1/Y1/X2/Y2）+ 轴向 1 通道（Z）              |
| 电流环      | PI 控制，每通道 A/C 双线圈，共 10 个 PI 控制器               |
| 位置环      | PID 控制，5 个 PID 控制器（Controller_Pos1~5）         |
| 双 CPU 架构 | CPU1 = 实时控制（电流/位置环），CPU2 = 故障监控/状态机           |
| 通信       | CPU1↔CPU2 通过 IPC 共享 RAM（`cla_logic_shared.h`） |

## 1️⃣ 主控侧：必须烧录程序 + 配置机型

### ✅ What（必须满足）

- 主控侧必须：
  - 烧录对应 **Sirius 软件**
  - 正确配置 **机型 / Variant**
- 否则：
  - Sirius 相关参数 **不会正确显示**
  - 后续调试没有意义

> ⚠️ 这一条**不在 MBC debug tool 内完成**，但它是所有后续步骤的**前置条件**

------

## 2️⃣ MBC 硬件侧：检查拨码，确保传感器真实可用

### ✅ What

- 在怀疑 **Pos1 / Pos4 / 传感器异常** 前
   👉 **必须先确认 MBC 板载拨码**

错误拨码会直接导致：

- 传感器被逻辑屏蔽 
- 表现为“无信号”

### ✅ How（操作要点）

- 物理检查 **MBC 板载拨码**
- 与机型 / 传感器配置表一致
- 若拨码错误：不进入校准 /不进入悬浮

> ✅ 这一步是**硬件前置检查**，不是 debug tool 步骤; 后续郭小俊输出的拨码文档，正是为了固化这一关

------

## 3️⃣ MBC 软件侧：连接 debug tool，确认通信正常

### ✅ What

- 确认 **debug tool ⇄ MBC** 通信是通的
- 否则所有参数操作都无效

### ✅ How（来自手册，逐步）

1. 打开 **debug tool（UDP_W5500）**
2. reload map 文件, 文件名：`bpp_main_cpu01`
3. 点击 **udp_open**
4. 观察曲线：若曲线开始波动 ✅ → 说明 tool 与 MBC 通信正常

------

## 4️⃣ MBC 校准：轴承校准 + 写入 FLASH（关键闭环）

### ✅ What（这是你强调的重点）

MBC **不只是跑一下**，而是必须完成：

1. ✅ 最新程序已烧录
2. ✅ 完整做一遍 **轴承校准**
3. ✅ 参数 **写入 FLASH**（不是只在 RAM）

------

### ✅ How（严格按手册执行）

#### Step 4.1 开始前准备

- **打开 record**
- 文件命名示例： 1761283649_10.24_calibration

------

#### Step 4.2 切换为 debug tool 控制

- 设置：
  - `PIC6 = 0`  → 从 PIC6 控制切换为 **debug tool 控制**
  - `Mode = 6`  → 进入 **轴承校准模式**

------

#### Step 4.3 观察校准过程

- 现象：电流（`ia / ic`）开始波动 /  位移曲线（`Pos_dis`）开始变化
- 正常参考范围：`Pos_dis1 ~ Pos_dis4 ≈ 100 ~ 150`

若： Pos 明显偏离 → **必须重新校准**

------

#### Step 4.4 写入 FLASH（不能省）

- 校准完成后：设置 `Mode = 267`  ✅ 将校准参数 **写入 MBC FLASH**
- 然后：关闭 record  ; 重命名记录文件

✅ 到这里，**MBC 校准闭环才算完成**

------

## 5️⃣ 悬浮自检（debug 模式）

### ✅ What

- 在进入正式开机前 👉 **必须至少完成一次悬浮自检**

------

### ✅ How（debug 操作）

1. 设置：`Mode = 5`（悬浮轴承）/  `Setspeed = 253`

2. 观察：位移是否悬浮到中心 / 波动是否 ≈ **10 μm**

3. 在 Mode=5 后 5 秒内：`InjAmp = 60` /  `InjBool = 677`

4. 观察 :  Cpu2_status ：= 2` ✅ 自检正常; `= 1` → 自检过慢; `= 3` ❌ 自检失败

------

## 6️⃣ 悬浮开机（PIC 操作模式）

### ✅ What

- debug 阶段完成后, 切换到 **PIC 控制开机**

------

### ✅ How

1. 设置：`Mode = 5`
2. 依次设置：`Setspeed = 251`（清故障）/ `Setspeed = 253`（报警模式）
3. 无异常 → 正常开机
4. 打开 record, 命名示例： 1761283649_10.24_16pm_running

------

## 7️⃣ 开机过程监控（安全红线）

### ✅ What

- 即使进入报警模式
- **并非所有异常都能被软件捕捉**

------

### ✅ How（人工监控要点）

- 重点关注：位移 **持续 > 120 μm**  / 电流 **> 10 A** / 异响 
- 任一出现：**立即停机**

## 4. 轴承校准（Mode=6）—— 完整参数手册

### 4.1 校准原理

Mode=6 执行正弦电流注入标定（`interface.c:639-706`）：

- 向每对线圈注入 `sin(2θ)` / `cos(2θ)` 电流
- 扫频范围：0 → 62.832 rad（10π，约 2×5 个周期）
- 前半程（0~31.416 rad）：激励前端径向 X1/Y1
- 后半程（31.416~62.832 rad）：激励后端径向 X2/Y2
- 轴向 Z 全程激励
- 记录每个传感器的 max/min，计算中心偏移：`pos_ref = 0.5 × (max + min)`

### 4.2 校准操作步骤 + 调参细节

| 步骤  | 操作           | 涉及参数                          | 默认值    | 源码位置                  | 调参说明                               |
| --- | ------------ | ----------------------------- | ------ | --------------------- | ---------------------------------- |
| 1   | 打开 record    | —                             | —      | —                     | 命名：`时间戳_日期_calibration`            |
| 2   | 切换控制权        | `PIC6 = 0`                    | 1111   | `main_cpu01.c:98`     | 必须先切到 debug tool                   |
| 3   | 进入校准         | `Mode = 6`                    | 31     | `main_cpu01.c:97`     | 进入前用 Mode=0 清零 PID 积分器             |
| 4   | **调整注入频率**   | `InjFre`                      | 0.2 Hz | `interface.c:101`     | 频率过高→传感器跟踪不上；过低→时间太长。范围 0.1~0.5 Hz |
| 5   | 观察校准过程       | `Pos_dis1~4`                  | —      | `main_cpu01.c:169`    | 正常范围 100~150 μm 振幅                 |
| 6   | 观察校准结果       | `pos_ref1~5`                  | —      | `interface.c:696-700` | 计算出的中心偏移                           |
| 7   | 验证极值         | `ref1_max/min ~ ref5_max/min` | —      | `interface.c:99-100`  | max-min 应在 50~400 之间，否则注入幅度或传感器有问题 |
| 8   | **写入 FLASH** | `Mode = 267`                  | —      | CPU2 Flash@0xc28000   | **必做！**                            |
| 9   | 关闭 record    | —                             | —      | —                     | —                                  |

### 4.3 校准隐藏参数

| 参数            | 含义                       | 默认值               | 源码                    | 何时需要调      |
| ------------- | ------------------------ | ----------------- | --------------------- | ---------- |
| `InjFre`      | 注入正弦频率 [Hz]              | 0.2               | `interface.c:101`     | 传感器响应慢时降低  |
| 注入幅度          | 硬编码 0.4 ± 0.4（占空比 0~0.8） | 0.8 peak          | `interface.c:645-658` | 需改代码       |
| `cal_theta`   | 相位累加器                    | 0.0               | `interface.c:102`     | Mode=0 可清零 |
| `pos_scale_p` | 位置传感器缩放                  | 试验台: 0.296 μm/LSB | `main_cpu01.c:390+`   | 换传感器必须校验   |
|               |                          |                   |                       |            |

## 5. PID 增益调整 —— 全轴参数对照表

### 5.1 位置环 PID（核心调参对象）

源码：`main_cpu01.c:127-140`，运行时 `set_para=1` 触发重载

**试验台配置（testrig=1，当前激活）：**

|轴|变量名|Kp|Ki|Kd|
|---|---|---|---|---|
|X1 径向-叶轮-X|`KP_d1, KI_d1, KD_d1`|**1.0**|1e-4|**0.003**|
|Y1 径向-叶轮-Y|`KP_d2, KI_d2, KD_d2`|**1.0**|1e-4|**0.003**|
|X2 径向-压缩机-X|`KP_d3, KI_d3, KD_d3`|**1.0**|1e-4|**0.003**|
|Y2 径向-压缩机-Y|`KP_d4, KI_d4, KD_d4`|**1.0**|1e-4|**0.003**|
|Z 轴向|`KP_d5, KI_d5, KD_d5`|**1.2**|1e-4|**0.009**|

**量产配置（testrig=0）：** 径向 Kp=0.5/Kd=0.0015，轴向 Kp=0.6/Kd=0.009

**PID 输出限幅：** `pid_limit = 1.0`（归一化 ±100%）

### 5.2 电流环 PI

|通道|Kp|Ki|源码|
|---|---|---|---|
|径向（X1~Y2）|**2.0**|**0.02**|`main_cpu01.c:149-150`|
|轴向（Z）|**0.7**|**0.06**|`main_cpu01.c:151-152`|

### 5.3 PID 调参操作

```txt
1. 修改 KP_dN / KI_dN / KD_dN
2. set_para = 1  → 触发 InitParameter()
3. 观察 Pos_dis1~5
4. 电流环: 改 KP_c / KI_c / KP_c5 / KI_c5
5. 重复直到位移波动 < 10 μm
```

### 5.4 调参经验法则

|现象|可能原因|调整方向|
|---|---|---|
|位置振荡不收敛|Kd 过大 或 Kp 过大|先降 Kd 50%，再降 Kp|
|稳态偏移大|Ki 太小|缓慢 ×2 增大 Ki|
|响应迟钝|Kp 太小|增 Kp，同时加 Kd 抑制过冲|
|电流震荡/啸叫|电流环 Kp 过大|降低 KP_c / KP_c5|
|轴向 micro-touch|Kd5 不够 / 补偿增益不足|增 KD_d5 和 `Axial_Compensation_Gain`|

## 6. 悬浮自检（Mode=5 + 注入诊断）—— 完整参数

### 6.1 基本悬浮

|步骤|参数|值|
|---|---|---|
|进入悬浮|`Mode`|**5**|
|启动运行|`Setspeed`|**253**|
|等 5 秒|—|等位移稳定|
|检查|`Pos_dis1~5`|< **10 μm** 波动|

### 6.2 注入信号自检参数（**基础版严重缺失的部分**）

|参数|含义|建议值|源码|调参说明|
|---|---|---|---|---|
|`InjBool`|注入使能|**677**|`main_cpu01.c:96`|特定编码启动|
|`InjAmp`|注入幅度|**60**|debug tool|过小→信噪比差；过大→位移超限|
|`InjFre`|注入频率|**0.2 Hz**|`interface.c:101`|自检可调到 0.5~2 Hz 测试频段响应|
|`KD_d1~5`|**微分增益**|见 §5.1|`main_cpu01.c:128-132`|**自检前可能需临时降 Kd**，避免对注入过度响应|
|`KP_d1~5`|比例增益|见 §5.1|`main_cpu01.c:128-132`|注入振幅超标时临时降 Kp|
|`Filter_Bool`|滤波器使能|**1**|`main_cpu01.c:101`|可关（=0）观察原始信号|

### 6.3 自检判定

|观察量|正常|进行中|失败|
|---|---|---|---|
|`Cpu2_status` / `drive_state`|**= 2** (RUNNING)|**= 1** (READY)|**= 3+** (FAULT)|
|`Pos_dis1~4` 径向|< 10 μm|收敛中|> 120 μm 持续|
|`Pos_dis5` 轴向|< 10 μm|收敛中|> 50 μm DC 偏移|
|`ia1~ia5, ic1~ic5`|< 5 A 稳态|正常波动|> 10 A 持续|

### 6.4 径向自检调参要点

```
若某通道振荡:
  → 降 KD_dN（0.003 → 0.0015），set_para=1
若偏移大:
  → 检查 pos_refN / refxNtem
  → 增大 KI_dN
径向注入测试:
  → InjBool=677, InjAmp=60
  → 轨迹半径 < 160 μm (WARNING) / < 200 μm (FAULT)
```

### 6.5 轴向自检调参要点

```
若振荡:
  → 降 KD_d5 (0.009→0.005→0.003), set_para=1
  → 降 KP_c5 (0.7→0.4)
若 DC 偏移大 (micro-touch):
  → 检查 Axial_DC_Mean
  → 增大 Axial_Compensation_Gain (0.001→0.003→0.005)
注入扫频:
  → InjFre = 0.1 → 0.2 → 0.5 → 1.0 → 2.0 Hz
  → 每频率 5~10 秒，记录 Pos_dis5 振幅
CPU2 三分量监控阈值:
  → DC 偏移 > 50 μm 报警
  → AC RMS > 30 μm 报警
  → 速度 RMS > 20 mm/s 报警
```

## 7. 轴向启动状态机（Mode=7）

### 7.1 状态机流程

```
IDLE(0) → PULL_IN(1) → SETTLING(2) → CLOSED_LOOP(3)
              │                │
              │ 超时500ms      │ 漂移>60counts
              ↓                ↓
          FAULT(4)        返回PULL_IN
```

### 7.2 可调参数

|参数|默认值|单位|源码|调参|
|---|---|---|---|---|
|`az_pull_current`|**7.0**|A|`interface.c:88`|过小→拉不动；过大→过冲|
|`az_hold_current`|**3.0**|A|`interface.c:89`|配合 pull 提供阻尼|
|`az_bias_current`|**3.5**|A|`interface.c:90`|闭环前缓冲|
|`az_pos_threshold`|**30.0**|ADC counts|`interface.c:91`|过小→难触发；过大→精度差|
|`az_pull_timeout`|**10000**|cycles(500ms)|`interface.c:92`|初始位置远时加大|
|`az_settle_time`|**2000**|cycles(100ms)|`interface.c:93`|增大让过渡更平稳|

### 7.3 故障排查

```
卡在 PULL_IN:
  → az_pull_current 7→9 A
  → az_pull_timeout 10000→20000 (1s)
SETTLING 反复回退:
  → az_pos_threshold 30→50
```

## 11. 典型调试场景

### 场景 A：首次上电全流程

```
PIC6=0 → Mode=0(清零) → Mode=6(校准, InjFre=0.2)
→ Mode=267(写FLASH) → Mode=5, Setspeed=253(悬浮)
→ InjBool=677, InjAmp=60(自检) → 观察 Cpu2_status=2
→ InjBool=0(停止注入)
```

### 场景 B：轴向振荡排查

```
1. Pos_dis5 > 30 μm → 降 KD_d5 (0.009→0.005), set_para=1
2. 高频振荡(>1kHz) → 降 KP_c5 (0.7→0.4)
3. DC 偏移 → 增 Axial_Compensation_Gain (0.001→0.003)
4. 持续异常 → 查传感器 AZ1_Jump_Fault / AZ2_Jump_Fault
```

### 场景 C：径向不稳

```
1. 检查 pos_refN 校准值
2. 降 KD_dN (×0.5), set_para=1
3. 电流震荡 → 降 KP_c
4. 观察 CPU2 RMS 半径: WARNING=160μm, FAULT=200μm
```

### 场景 D：自检注入扫频

```
Mode=5, Setspeed=253, InjBool=677
InjFre: 0.1→0.2→0.5→1.0→2.0 Hz
每频率记录 Pos_dis 和 ia 响应
异常大振幅 → 共振频率 → 考虑 notch filter
临时降 KD 减少微分放大
```
