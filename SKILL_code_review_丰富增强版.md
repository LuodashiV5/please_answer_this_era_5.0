# SKILL: Code Review（丰富增强版）

> 给 Claude Code 使用：对我贴出的嵌入式 C/C++ 代码做“尽可能完整”的 review（包含 MISRA/CERT/安全/可维护性），但输出依旧要直接、中文旁注、只提改善建议。

## 目标
- **第一优先：致命风险**（越界/溢出/空指针/竞态/死锁/UB/硬件误动作）
- **第二优先：合规与质量**（MISRA C:2012、CERT C、内部规范、可测试性）
- **第三优先：工程落地**（给出最小修改补丁 + 如果需要，给“理想重构方向”）

## 输出（必须严格遵守）
1) 只输出改善建议，不要讲过程。
2) **逐行中文旁注**：
   - `// REVIEW[BLOCKER]: ... (MISRA: Rxx.x)`
   - `// REVIEW[MAJOR]: ... (CERT: XXX)`
   - `// REVIEW[MINOR]: ...`
3) 发现规则冲突或信息不足时：
   - 允许标注 `// REVIEW[MAJOR]: ... (待确认：缺少类型/宏定义/上下文)`
4) **必须提供补丁**：
   - 先给“最小修复补丁”
   - 再给“可选增强补丁”（如将宏替换为 inline、拆分复杂条件、增加单测点）
5) **输出结构**（必须按顺序）：
   - A. 代码旁注（直接给标注后的代码）
   - B. 补丁（最小修复）
   - C. 可选增强（如果有）

## 特别关注点（嵌入式/控制项目常见坑）
- 宏：多次求值、优先级、参数副作用、类型不一致
- 数值：位移超宽、signed/unsigned 混算、截断、溢出、float/double 常量
- 并发：ISR 与任务共享变量、读改写原子性、多核共享 RAM
- 外设/协议：长度字段可信度、CRC/校验、帧解析越界、DMA 缓冲区一致性
