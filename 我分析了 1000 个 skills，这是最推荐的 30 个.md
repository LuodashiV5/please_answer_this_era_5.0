**01**

去年 Prompt 最火的时候，我做了一个提示词共享平台PrompterHub.cn。半年多过去，平台积累了一万用户。

在 Skill 刚火起来的时候，我不只一次想在这个平台加入 Skill 模块，但Skill 是托管文件夹，Prompt 是处理文本，前者对技术和资源要求高，靠我一人很难做起来。

不过 openclaw 的创始人 Peter Steinberger 就做了这个平台 **ClawHub**，一个面向 AI Agent 的 Skills 共享平台，目前平台上已有**10,322 个 Skill**，涵盖了从企业 SaaS 到个人生产力的各个领域。

![飞书文档 - 图片](https://mmbiz.qpic.cn/mmbiz_png/CFe2b8yvCozPUMZub4rTC34ZHyJ87gnHGvPRJZFEszh6IDPkxX5tn3pV7boaJVeicmeI26Xzo8y2qBvwr8UckhK0ic6AYryiawCibzhWl10ho88/640?wx_fmt=png&from=appmsg)

因为 openclaw 开源爆火的缘故，这个平台也成了最权威的 Skill 网站之一。

比如以前你想要把 YouTube 视频转成中文总结，得先找到合适的 Prompt，复制粘贴给 AI，再发视频链接，每次都要重复这套流程。但现在有了 Skill，你只需要装一次「YouTube 视频转中文」的 Skill，之后直接丢 Youtube 链接就行，AI 自动识别、自动处理、自动输出。

也可以到**ClawHub**网站探索一下，看看别人是怎么做Skill 的。

我花了几天时间，分析了下载量前 1000 的 Skill，想弄明白一个问题：**Skill 究竟可以应用到哪些领域？哪些场景已经被验证？未来 Skill 会不会终结传统的 SaaS 模式？**

**02**

如果把 AI Agent 比作智能手机，那**ClawHub 就是它的 App Store**。

![飞书文档 - 图片](https://mmbiz.qpic.cn/sz_mmbiz_png/CFe2b8yvCowcSnNRRAfpFviaUYE5SLzVqS9yRLjWL9orXtCyzrwaiaxJ0OfpbPap2fEyfQbM9sWwllicicVlcynfZ6ibypfIg2YbokAYicA3V6pQM/640?wx_fmt=png&from=appmsg)

简单来说，ClawHub 提供三件事：

**发布与版本化**。开发者可以将自己编写的 Skill 上传到 ClawHub，像发布 App 一样进行版本管理。用户安装时能明确知道这是 1.0 还是 2.0，更新日志写了什么。

**检索与发现**。目前平台已有**10,322 个 Skill**，涵盖办公、开发、内容创作、金融等各个领域。用户可以通过分类、搜索、排行榜找到需要的 Skill。

**ClawHub 是 Skill 的仓库，OpenClaw 是运行 Agent 的平台**。

这种分工让 Skill 生态有了专业的基础设施。开发者专注写 Skill，用户专注用 Skill，平台负责连接双方。

**03**

我分析了 ClawHub 下载量前 1000 的 Skill，发现它们可以归纳为 6 大领域。每个领域的分布比例和核心逻辑，反映了当前 AI Agent 落地的真实需求。

Agent 需要「手和脚」来打破数据孤岛，参与到真实的业务流转中。这是目前绝对的主流刚需。

这类 Skill 主打「替人跑腿」，让 Agent 深度介入真实业务系统。Google Workspace、Slack、Notion 等办公协同巨头占据统治地位；Zoho、Salesforce、HubSpot 等业务系统形成密集矩阵；API Gateway、n8n 等自动化中间件则扮演连接一切的「胶水」角色。

**如果你在开发 B 端系统，也可以学习 Notion、Slack 一样，尽早开放 API，为 AI Agent 时代做好准备。**

**04**

ClawHub 的 Skill 使用非常简单，目前支持两种方式。

![飞书文档 - 图片](https://mmbiz.qpic.cn/sz_mmbiz_png/CFe2b8yvCoyUjG7FWPeYsYwSWMGZctIandLvDL9ibPn6Vb7tQXibKsLzDWicETb2W5YFgJcvXYhSJX22blWsMfsIslXVD4SSYrkySF3Vicdbu1s/640?wx_fmt=png&from=appmsg)

**1 命令行安装（OpenClaw 用户）**

如果你在使用 OpenClaw，可以直接通过命令行安装：

```
npx clawhub@latest install sonoscli
```

这种方式默认安装到`openclaw/skills`目录下，支持版本管理和回滚。安装后，Skill 立即生效，无需额外配置。

**2 对话安装（Claude Code 用户）**

如果你在用 Claude Code，安装更简单：

- 直接发给 Claude：「安装这个 Skill」
    
- Claude 会自动下载到`.claude/skills`目录并启用
    

**技巧：软链接同步**

如果你同时用 OpenClaw 和 Claude Code，可以用软链接让两个平台共享同一套 Skill：

```
ln -s ~/.claude/skills ~/openclaw/skills
```

这样在一个平台安装的 Skill，另一个平台也能直接用。

**3 从 GitHub 导入**

clawhub 还支持直接从 GitHub 仓库导入 Skill。只要仓库里有标准的`skill.json`配置文件，ClawHub 就能自动识别并安装。这比上传文件方便很多。

**05**

做完这个分析，我有一个越来越强烈的判断：**Skill 正在吃掉 SaaS，**或者说是订阅制的软件，尤其是工具类。

SaaS 不会消失，但它的使用方式和商业模式会被改变。

![飞书文档 - 图片](https://mmbiz.qpic.cn/mmbiz_png/CFe2b8yvCoxRwt7VGo06JlWeK5ogJnHoQ8sYjnaq2EicxdWjdb1cDKMvib0UkkBj7VrJcvib3tjpJKib0rA6juY3JWyfgeymbQYDCv9TlPZFKFk/640?wx_fmt=png&from=appmsg)

**从「打开应用」到「直接说话」**

传统 SaaS：你想记录想法，先打开笔记 App，打字输入完成。

Skill 模式：直接跟 AI 输入笔记，Agent 调用 笔记记录 Skill ，保存内容并同步到你的各个笔记软件里，甚至能根据你的文风，从笔记扩写一篇文章，制作一个视频等等。

**从「订阅制」到「按需调用」**

传统 SaaS：每月固定付费，不管你用不用。Notion、Slack、各种工具堆在一起，月费几百块。

Skill 模式：按调用付费，甚至免费。

**从「功能堆砌」到「原子化能力」**

传统 SaaS：每个产品都想做大而全，功能越堆越多，越来越臃肿。

Skill 模式：每个 Skill 只做一件事，但做到极致。Gog 只专注 Google Workspace 控制，Summarize 只专注内容提炼。

**从「数据孤岛」到「上下文共享」**

传统 SaaS：你在 Notion 写的文档，在 Slack 讨论的内容，在 Gmail 发的邮件，彼此割裂。

Skill 模式：Ontology Skill 构建知识图谱，让不同 Skill 共享上下文状态，形成统一的工作记忆。

未来的 SaaS 可能更像「基础设施提供商」，提供 API 和底层能力，而 Skill 开发者负责把这些能力封装成用户友好的交互方式。

就像 AWS 不会消失，但用户不会直接操作 AWS 控制台，而是通过 Skill 间接调用它的计算资源。
